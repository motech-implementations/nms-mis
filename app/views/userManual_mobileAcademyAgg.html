MA Cumulative Summary Report


This report gives a glance of the performance of the Mobile Academy project from start date of the project to till date. It gives information about the number of ASHAs registered with the program, the number of ASHAs that have started the course, the number of ASHAs who have not yet started the course, the number of ASHAs who have passed the course and the number of ASHAs who have failed the course. All the above counts are for ASHAs with job status ‘ACTIVE’. This report will give a snapshot of how far each State / UT / district is on the aspect of training all the ASHAs. This report will primarily show the data for the first-time training and completion of the course of the ASHAs.

This report will help to represent the overall performance of course completion by ASHA in selected state.

The report displays the data in the following sequence of columns.

  1. Serial Number
  2. State
  3. No of Registered ASHA
  4. No of ASHA Started Course
  5. No of ASHA Not Started Course
  6. No of ASHA Successfully Completed the Course
  7. No of ASHA who failed the course
  8. % Not Started Course
  9. % Successfully Completed
  10. % Failed the course

The rules governing the display components of the report is given below

  1. Columns 6 & 9 should take into consideration only the first successful completion and not repeat attempts.
  2. The percentage displayed in the report should follow the below rules based on data in its respective rows only
    1. % Not Started Course: (Column 5/Column 3) * 100
    2. % Successfully Completed: (Column 6/Column 4) * 100
    3. % Failed the course: (Column 7/Column 4) * 100

The report should display data based on the As-On date selected with Registration, Started, Not Started, Completed and Failed data based on cumulative since the program was started.

1) The users can login into the system using the URL: https://rchivrreports.in/.

2) Under "Reports Category" select "Mobile Academy Reports".




3) Under "Reports" select "Cummulative Summary"




4) Select an ‘End Date’ for which the cumulative summary report is required.




5) Click on ‘Submit’




6) The report will be generated




7) Once the report is generated, it can be downloaded in PDF, CSV and XLSX formats.




MA Subscriber Report


This report gives the glance of the subscriber load on the Mobile Academy project. It shows how many ASHAs who have registered but not completed the course at the start of the period, the number of ASHAs who have joined the course and number of who have completed the course. All the above counts are for ASHAs with job status ‘ACTIVE’.

The report displays the data in the following sequence of columns

  1. Serial Number
  2. State
  3. No of ASHA Registered But Not Completed (Period Start)
  4. No of ASHA Records Received Through Web Service
  5. No of ASHA Records Rejected
  6. No of ASHA Records Added
  7. No of ASHA Successfully Completed the Course
  8. No of ASHA Registered But Not Completed (Period End)

The rules governing the display components of the report is given below

  1. The data pertaining to column 3, 4, 5, 6, 7, 8 should be for the selected period only.
    1. Column 3 will display the number of ASHAs who have registered in the MA course prior to the start of the period but have not completed the course.
    2. Column 4 will display the number of ASHA Records that have been received from web service from MCTS/RCH during the period.
    3. Column 5 will display the number of the records that have been rejected due to wrong mobile number and other reasons – a subset of (b).
    4. Column 6 will display the number of ASHAs who have been added/subscribed in MA course for the first time in the selected period. Column 6 = Column 4 – Column 5.
    5. Column 7 will display the number of ASHAs who have successfully completed the course for the first time, during the selected period and secured pass marks.
    6. Column 8 will display calculation of Column 3 + Column 6 – Column 7. This number will reflect the number of ASHAs presently Subscribed in the Mobile Academy program but are yet to start or complete the course.

1) The users can login into the system using the URL: https://rchivrreports.in/.

2) Under "Reports Category" select "Mobile Academy Reports", then




3) Under "Reports" select "Subscriber Report"




4) Select ‘Year’ under ‘Period Type’.




5) Select the year for which report is needed.




6) Click on ‘Submit’.




7) The report will be generated.




8) Once report is generated, it can be downloaded in PDF, CSV or XLSX formats.




9) Select ‘Financial Year’ under ‘Period Type’ and choose the start year for which report is needed.




10) Click on ‘Submit’ to generate the report




11) The report will be generated which can be downloaded in PDF, CSV or XLSX formats.

12) Select ‘Quarter’ in the ‘Period Type’ and select the year and quarter for which the report is needed. Then click ‘Submit’.




13) The generated report can be downloaded in PDF, CSV and XLSX formats.

14) Select ‘Month’ under ‘Period type’ and choose a month and click ‘Submit’.




15) The report will be generated for the chosen month and can be downloaded in PDF, CSV and XLSX formats.

16) ‘Custom Range’ under ‘Period Type’ can also be selected. After that, you need to select a start date and an end date for which report is needed, then click on ‘Submit’




17) The report will be generated for the specified date range, which can be downloaded in PDF, CSV and XLSX formats.

MA Performance Report


This report gives at a glance the performance of the Mobile Academy Project for the selected period. It gives information about how many ASHAs have started the course in the period, number of ASHAs who have accessed the course during the period, number of ASHAs who had started the course previously but have not listened to any content during the period, number of ASHAs who have completed the course successfully in the selected period and those who have completed the course in the selected period but could not secure pass marks. All the above counts are for ASHAs with job status ‘ACTIVE’.

The report displays the data in the following sequence of columns

  1. Serial Number
  2. State
  3. No of ASHA Started Course
  4. No of ASHA Pursuing Course
  5. No of ASHA Not Pursuing Course
  6. No of ASHA Successfully Completed the Course
  7. No of ASHA who failed the course

The rules governing the display components of the report is given below

  1. The data pertaining to column 3, 4, 5, 6 and 7 should be for the selected period only.
    1. Column 3 will display the number of ASHAs who have started the course for the first time in the selected period.
    2. Column 4 will display the number of ASHAs who had started the course before the selected period, had accessed the course at least once with one bookmark during the selected period and not completed the course till the end of the selected period (Please refer to the NOTE).
    3. Column 5 will display the number of ASHAs who had started the course before the selected period and had not accessed the course once during the selected period. The count does NOT include ASHAs who have any successful completion till the end of the selected period (Please refer to the NOTE).
    4. Column 6 will display the number of ASHAs who have successfully completed the course for first time, during the selected period and secured pass marks.
    5. Column 7 will display the number of ASHAs who have completed the course during the selected period and did not secure passing marks even once till the end of selected period.

NOTE Related to Columns 4 and 5 (Persuing and Not-Persuing):

These two metrics are not pregenerated. So they are based on live data while runtime, i.e., at the time when user is requesting for the report via MIS Portal.

Reason: For any report metric, the cumulative data upto any given date is available. For a selected date range, we pick the required metric as on these two dates and their difference will provide the required value. For example, number of first successful completion records.

But in case of pursuing and not pursuing, the calculation is based two dates. We can’t have a cumulative figure “count of pursuing as of a particular date”. So we cannot pregenerate this for every startDate - endDate combination.

1) The users can login into the system using the URL: https://rchivrreports.in/.

2) Under "Reports Category" select "Mobile Academy Reports", then




3) Under "Reports" select "Performance Report"




4) Select ‘Year’ under ‘Period Type’.




5) Select the year for which report is needed.




6) Click on ‘Submit’.




7) The report will be generated.




8) Once report is generated, it can be downloaded in PDF, CSV or XLSX formats.




9) Select ‘Financial Year’ under ‘Period Type’ and choose the start year for which report is needed.




10) Click on ‘Submit’ to generate the report




11) The report will be generated which can be downloaded in PDF, CSV or XLSX formats.

12) Select ‘Quarter’ in the ‘Period Type’ and select the year and quarter for which the report is needed. Then click ‘Submit’.




13) The generated report can be downloaded in PDF, CSV and XLSX formats.

14) Select ‘Month’ under ‘Period type’ and choose a month and click ‘Submit’.




15) The report will be generated for the chosen month and can be downloaded in PDF, CSV and XLSX formats.

16) ‘Custom Range’ under ‘Period Type’ can also be selected. After that, you need to select a start date and an end date for which report is needed, then click on ‘Submit’




17) The report will be generated for the specified date range, which can be downloaded in PDF, CSV and XLSX formats.

MA Performance Report


This report gives at a glance the performance of the Mobile Academy Project for the selected period. It gives information about how many ASHAs have started the course in the period, number of ASHAs who have accessed the course during the period, number of ASHAs who had started the course previously but have not listened to any content during the period, number of ASHAs who have completed the course successfully in the selected period and those who have completed the course in the selected period but could not secure pass marks. All the above counts are for ASHAs with job status ‘ACTIVE’.

The report displays the data in the following sequence of columns

  1. Serial Number
  2. State
  3. No of ASHA Started Course
  4. No of ASHA Pursuing Course
  5. No of ASHA Not Pursuing Course
  6. No of ASHA Successfully Completed the Course
  7. No of ASHA who failed the course

The rules governing the display components of the report is given below

  1. The data pertaining to column 3, 4, 5, 6 and 7 should be for the selected period only.
    1. Column 3 will display the number of ASHAs who have started the course for the first time in the selected period.
    2. Column 4 will display the number of ASHAs who had started the course before the selected period, had accessed the course at least once with one bookmark during the selected period and not completed the course till the end of the selected period (Please refer to the NOTE).
    3. Column 5 will display the number of ASHAs who had started the course before the selected period and had not accessed the course once during the selected period. The count does NOT include ASHAs who have any successful completion till the end of the selected period (Please refer to the NOTE).
    4. Column 6 will display the number of ASHAs who have successfully completed the course for first time, during the selected period and secured pass marks.
    5. Column 7 will display the number of ASHAs who have completed the course during the selected period and did not secure passing marks even once till the end of selected period.

NOTE Related to Columns 4 and 5 (Persuing and Not-Persuing):

These two metrics are not pregenerated. So they are based on live data while runtime, i.e., at the time when user is requesting for the report via MIS Portal.

Reason: For any report metric, the cumulative data upto any given date is available. For a selected date range, we pick the required metric as on these two dates and their difference will provide the required value. For example, number of first successful completion records.

But in case of pursuing and not pursuing, the calculation is based two dates. We can’t have a cumulative figure “count of pursuing as of a particular date”. So we cannot pregenerate this for every startDate - endDate combination.
