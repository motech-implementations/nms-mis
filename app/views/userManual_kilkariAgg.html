Kilkari Repeat Listener Month-wise Report


This report gives month-wise distribution of all beneficiaries of a defined geographic zone, in terms of the number of calls successfully answered during the specified month; i.e. this report displays a count of beneficiaries who have listened to more than 5 calls, 5-calls, 4-calls, 3-calls, 2-calls, 1 call and 0 calls. This report gives a clear picture of the percentage of regular listeners versus those who listen less regularly. This report gives the details of the count month by month for the period of 6 months.

This report is generated by the fifth of every month for the previous month giving details of the month by month calls for the last six months. The report has two tables – the first one gives the count and the second giving the relative percentages.

Drill-down from State to District to Block and Sub-center cannot be supported for this report as there is no corresponding column in the matrix/table.

The rules governing this report as stated below:

  1. The logic of the data displayed in the report is as follows
    1. The report displays the data for 6 months from either the launch of Kilkari service in the region or December 2016.
    2. Some beneficiaries could have listened to more than 5 calls in case of network fluctuations in the backend system or telecom network.
    3. The data of the first row and the first column in the first grid are the count of beneficiaries who have listened to total of more than 5 calls in a particular calendar month.
    4. The data of the first row and the first column in the second grid are the percentage of beneficiaries who have listened to total more than of 5 calls in a particular calendar month among all the beneficiaries who have been called.
    5. The data under the column 0 calls answered is the number of beneficiaries who had been called during the month but did not answered/picked any of the calls made.
  2. The report displays data in the two grids – the first grid displays the actual numbers of beneficiaries.
  3. The second grid displays data in percentages (as a percentage of the total active beneficiaries in the geographic region to whom calls were attempted during the month) with the total summation for all columns for a particular month being 100.

The report presents the listening behavior of all active users in that period, i.e. all users to whom calls were attempted by the Kilkari system during the month. The data in the Total row should be summation of the data displayed in the four rows of the respective column. The data in the Total column is summation of data displayed in the four columns in the respective row.

Kilkari Message Matrix Report


This report gives two–dimensional matrix view displaying count of beneficiaries who have listened to average content duration (in bucketised content duration categories) across all the bucketised message-number categories. This report will give an understanding of the listening profile of subscribers as they age within the system - do the listeners show a trend of listening to more or less content as they hear more and more messages?

This report allows selection of calendar year, financial year, quarter, month and week, but does not allow custom date range selection. This is because, the distribution of beneficiaries needs to be pre-calculated and cannot be calculated in real-time.

This drill down option is not available for this report.

This report will display a matrix that will display the listening profiles of the beneficiaries. The rules governing this report are as stated below:

  1. The logic of the data displayed in the report is as follows
    1. The data in the grid of the first row and the first column is the count of beneficiaries who have listened to average of > 75% content across Message no 1 to 6. If a beneficiary has not listened to one or more message(s) from among message no. 1-6, the system will treat that as 0% of the content for that message listened to, and average content listened to across all 1-6 messages will be calculated based on that.
  2. The report displays the data in the following sequence of columns:
    1. Listened to >75% of the content
    2. Listened to 50 to 75% of the content
    3. Listened to 25 to 50% of the content
    4. Listened to <25% of the content
    5. Total
  3. The report displays the data in the following sequence for the mother Pack:
    1. Message Week 1-6
    2. Message Week 7-12
    3. Message Week 13-18
    4. Message Week 19-24
  4. The report displays the data in the following sequence for the child Pack:
    1. Message Week 1-6
    2. Message Week 7-12
    3. Message Week 13-18
    4. Message Week 19-24
    5. Message Week 25-30
    6. Message Week 31-36
    7. Message Week 37-42
    8. Message Week 43-48

The following are the rules governing the report:

  1. The following is the equation used for calculating the percentage content listened to for each beneficiary: % Content listened to = (Content listened to in all calls * 100) / (Sum or total content duration of all answered calls) Content duration and not on call duration is used as the basis of calculation in this report, as full content duration of each message is fixed, while the total expected call duration cannot be fixed.
  2. The bracket of ‘listened to <25% content’ will include beneficiaries who have not listened to any content, in case they answer the call.
  3. There is no ‘Total’ row for this report as a beneficiary could fall in two brackets in a selected period. For example, a beneficiary could have listened to message 5-6 and listened to 75% and then listened to message 7-8 and listened to 60% of content in the same month. So, total count of all beneficiaries across all categories would consider this beneficiary twice; thereby a ‘Total’ row for this report is meaningless.

    2. The rules governing the display components of the report is given below

    1. The following is the equation used for calculating the percentage calls answered for each beneficiary % Calls answered = (Successful Calls * 100) / (Success Calls + Failure Campaigns) in the selected period.

    2. The following is the equation used for calculating the percentage listened to for each beneficiary % Content listened to = (Actual content listened to in all answered calls * 100) / (Sum of total content duration of answered calls)
